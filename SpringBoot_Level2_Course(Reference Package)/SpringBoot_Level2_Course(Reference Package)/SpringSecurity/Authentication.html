In-Memory User Authentication

Creating In-Memory Users

@Configuration @EnableWebSecurity public class SecurityConfig { @Bean public UserDetailsService userDetailsService() { UserDetails user = User.builder() .username("user") .password(passwordEncoder().encode("password")) .roles("USER") .build(); UserDetails admin = User.builder() .username("admin") .password(passwordEncoder().encode("admin")) .roles("USER", "ADMIN") .build(); return new InMemoryUserDetailsManager(user, admin); } @Bean public PasswordEncoder passwordEncoder() { return new BCryptPasswordEncoder(); } }

User Roles and Authorities

Spring Security distinguishes between roles and authorities:

  • Roles: High-level permissions (ROLE_USER, ROLE_ADMIN)
  • Authorities: Fine-grained permissions (READ_USERS, WRITE_USERS)
