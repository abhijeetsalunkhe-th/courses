Core Security Concepts

Authentication vs Authorization

Concept Question Purpose Example
Authentication "Who are you?" Verify user identity Username/password, JWT token
Authorization "What can you do?" Grant permissions based on identity ROLE_ADMIN can delete users

Spring Security Architecture

Spring Security uses a chain of filters to process security concerns:

  • SecurityFilterChain: Defines security configuration for requests
  • AuthenticationManager: Processes authentication requests
  • UserDetailsService: Loads user-specific data
  • PasswordEncoder: Encodes and validates passwords

Key Security Annotations

@EnableWebSecurity // Enable Spring Security @EnableMethodSecurity // Enable method-level security @PreAuthorize("hasRole('ADMIN')") // Check before method execution @PostAuthorize("returnObject.owner == authentication.name") // Check after @Secured("ROLE_USER") // Simple role check @RolesAllowed("ADMIN") // JSR-250 annotation

Common Security Patterns

  • Stateless Authentication: JWT tokens for REST APIs
  • Session-based: Traditional web applications
  • OAuth2/OpenID Connect: Third-party authentication
  • Basic Authentication: Simple username/password
