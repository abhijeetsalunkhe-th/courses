What Is Extended Thinking?

Overview

Extended Thinking is a sophisticated capability that allows Claude to engage in a dedicated reasoning phase before producing its final output. Unlike standard responses where the model generates text sequentially, Extended Thinking provides a "private" workspace for the model to work through complex logic, evaluate multiple hypotheses, and self-correct.

User Input API Request with Thinking Enabled
➜
Thinking Phase Internal reasoning tokens (hidden from final output)
➜
Final Response Refined, accurate answer based on reasoning

The Mechanics of Thinking

  • Thinking Tokens: These are internal tokens used by the model during the reasoning phase. They are billed at the same rate as output tokens but are distinct from the final visible response.
  • Budget Control: Developers can set a budget_tokens parameter to control how much "thought" Claude is allowed to use for a given request.
  • Transparency: While the thinking process is internal, the API returns it in a specific thinking block, allowing developers to monitor or display the model's reasoning process if desired.

API Implementation

To enable Extended Thinking, you must use a supported model (e.g., Claude 3.7 Sonnet) and include the thinking configuration in your API call.

import anthropic

client = anthropic.Anthropic()

response = client.messages.create(
    model="claude-3-7-sonnet-20250219",
    max_tokens=20000,
    thinking={
        "type": "enabled",
        "budget_tokens": 8000
    },
    messages=[
        {"role": "user", "content": "Analyze this complex legacy codebase for potential race conditions..."}
    ]
)

# The response contains both thinking and text blocks
for block in response.content:
    if block.type == "thinking":
        print(f"Claude's Thoughts: {block.thinking}")
    elif block.type == "text":
        print(f"Final Answer: {block.text}")

When to Leverage Extended Thinking

Extended Thinking is most effective for tasks that require deep analysis, multi-step planning, or complex problem-solving. It is not necessary for simple tasks where the answer is straightforward.

Use Case Category Example Task Why use Extended Thinking?
Complex Coding Refactoring large functions Allows Claude to map dependencies and edge cases before writing code.
Mathematical Reasoning Solving multi-stage proofs Ensures each logical step is verified before proceeding to the next.
Security Analysis Vulnerability research Enables thorough exploration of potential attack vectors and exploit paths.
Architectural Design System scalability planning Facilitates the evaluation of multiple architectural trade-offs.

Key Characteristics

  • Chain of Thought: The model inherently uses Chain-of-Thought reasoning without needing explicit "think step-by-step" prompts.
  • Self-Correction: Claude can identify errors in its own reasoning during the thinking phase and adjust its final response accordingly.
  • Consistency: Leads to more consistent performance on high-difficulty benchmarks.
