RAG - Adding External Knowledge

What It Is

RAG augments an LLM by retrieving relevant documents at query time and including them in the prompt. The model's weights remain unchanged, but it has access to external knowledge.

Typical RAG Stack

ComponentPurposeExamples
Document StoreHold source documentsS3, GCS, database
Embedding ModelConvert text to vectorsOpenAI Ada, Cohere, BGE
Vector DatabaseSimilarity searchPinecone, Weaviate, pgvector
LLMGenerate answers from contextClaude, GPT-4, Gemini

Strengths

  • Current information: Add or update documents without retraining
  • Source attribution: Can cite specific documents and passages
  • Domain knowledge: Works with proprietary/internal data
  • Reduced hallucination: Grounds responses in retrieved facts

Limitations

  • Retrieval quality: Answer quality depends on finding the right documents
  • Latency: Additional retrieval step adds response time
  • Chunking challenges: Poor chunking leads to lost context
  • Infrastructure cost: Vector database hosting and embedding generation
