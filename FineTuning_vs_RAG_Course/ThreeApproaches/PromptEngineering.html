Prompt Engineering - The First Line

What It Is

Prompt engineering is the art of crafting input text to get desired outputs from a pre-trained model without any modification. The model's weights remain unchanged.

Techniques

  • System prompts: Define role, constraints, and output format
  • Few-shot examples: Show desired input-output patterns
  • Chain of thought: Guide step-by-step reasoning
  • XML/structured tags: Organize complex prompts

Strengths

  • Zero cost: No training, no infrastructure beyond API calls
  • Instant iteration: Change the prompt and test immediately
  • No data needed: Works with zero training examples
  • Model-agnostic: Techniques work across providers

Limitations

  • Context window limits: Can only provide so much context per request
  • No knowledge injection: Cannot teach the model new facts
  • Inconsistency: Complex prompts may produce variable results
  • Per-request cost: Long system prompts increase token costs on every call
