SSO, Security, and Compliance

Identity Management with SAML 2.0

Enterprise Claude supports SAML 2.0, allowing organizations to centralize authentication through their existing Identity Provider (IdP). This ensures that login policies, such as Multi-Factor Authentication (MFA), are enforced.

Supported IdP Examples

  • Okta: Native app integration available in the Okta Integration Network.
  • Azure AD: Seamless integration using Enterprise Applications in the Azure Portal.
  • Google Workspace: Using Google as the SAML IdP.

Automated User Provisioning (SCIM)

System for Cross-domain Identity Management (SCIM) allows you to automate the lifecycle of user accounts. When a user is added to a specific group in your IdP, they are automatically provisioned in Claude. Conversely, when they leave the company, their access is instantly revoked.

Compliance and Certifications

Anthropic maintains rigorous standards to meet global enterprise requirements:

  • SOC 2 Type II: Independent audit of security, availability, and confidentiality.
  • HIPAA: Support for healthcare data via Business Associate Agreements (BAA).
  • GDPR: Comprehensive data processing agreements for EU users.

Scenario: Handling a Data Breach Protocol

Emergency Revocation: In the event of a suspected account compromise, an Org Admin can instantly terminate all active sessions for a user and disable their account through the Admin Console, overriding any cached IdP tokens.
